# 🏥 Fullerton Hospital System

Design-focused hospital management system modeling complex healthcare workflows using UML, object-oriented design, and GRASP principles. Emphasizes scalable architecture and clear separation of responsibilities across multiple subsystems.

---

## 📌 Overview

The Fullerton Hospital System is a comprehensive system design project that models a centralized platform for managing hospital operations. It supports multiple stakeholders—including patients, healthcare providers, administrators, and insurance companies—by integrating key functionalities such as patient records, appointment scheduling, billing, and inventory management.

This project focuses on applying software engineering principles to design a scalable and maintainable system using UML diagrams, object-oriented design, and GRASP principles.

---

## 🧠 Key Concepts

- Object-Oriented Design (OOD)  
- GRASP Principles (Controller, Information Expert, Low Coupling, etc.)  
- UML Modeling (Domain, Use Case, Sequence, SSD)  
- Layered Architecture (UI, Domain, Technical)  
- System Design for Complex, Multi-User Environments  

---

## 🏗️ System Design Highlights

- Designed a modular hospital system supporting multiple subsystems:
  - Patient Management  
  - Appointment Scheduling  
  - Billing & Insurance  
  - Inventory & Pharmacy  
- Modeled real-world healthcare workflows using UML diagrams  
- Applied GRASP principles to ensure maintainability and scalability  
- Structured the system using a layered architecture approach  

---

## 📊 Key Diagrams

### 1. Domain Model Diagram
Represents the core entities and relationships within the system, including patients, staff, appointments, billing, and inventory.

### 2. Use Case Diagram
Illustrates interactions between stakeholders (patients, doctors, administrators, insurance) and the system.

### 3. Sequence Diagrams
Show step-by-step workflows such as:
- Patient visits  
- Prescription handling  
- Billing processes  

### 4. System Sequence Diagrams (SSD)
Provide a high-level view of system operations and interactions between the user interface and backend logic.
